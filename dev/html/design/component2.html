Encore DPP DIA Digital Resource Guide (DRG)

Design Principles

Philosophy

Keeping with Encore's principles, the design for the Consumer Portal Pages and related tools will follow a philosophy of structure, simplicity, and consistency.

Our goal is to create simple, clean, intuitive, and friendly sites. We should visually guide the user by a combination of familiar interface and clean design.



We use Google's Material Design principles as a baseline or reference for our design. The objective is not to implement a pure Material Design user experience, but to use some of its core concepts in our unique design language and user experience.

Principles

Whenever applicable, we should follow these principles when designing a new page or UI component:

Consistent and Familiar Interface

  • Familiar: Use conventions and UI elements that are common in the world's most popular websites, so it is intuitive and easy for users to pick up.
  • Consistent: Use consistent interface throughout the whole site, so the users don't need to relearn how to use each page.

Visibility and User Focus

By using colors and contrast, our UIs should be bold yet clean, powerful yet friendly - keeping user focus on what is most important.

  • Contrast and color.
  • Large and readable fonts in every screen size.
  • Use Material Design cues.

Instant Feedback

  • Use motion and transformations to provide clear and subtle feedback without interrupting continuity.
  • Inform users of results of actions and state changes immediately.
    • Hover states.
    • Appear/disappear transitions.
    • Loading animations
    • Confirmation popups for risky actions.

Clear Structure

Our interfaces should follow a visual hierarchy in order to clearly guide the user through the flow.

  • Correct use of heading hierarchy (H1, H2, H3...).
  • Use white (negative) space to separate unrelated items.
  • Standardized style for tabular data.

Simplicity

  • Keep our UIs simple above everything else.
  • Do not include unnecessary elements.
  • Implement responsive design only when it is needed.
